## 1. B站评论区 Shadow DOM 调试

### 问题
尝试用 JS 隐藏置顶评论中的子评论和底栏按钮（点赞/回复等），多次失败。

### 尝试过的方案
| 方案 | 结果 |
|------|------|
| `page.evaluate()` + `querySelectorAll('*')` 在shadow root内递归隐藏 | `querySelectorAll` 不穿透嵌套 shadow DOM，按钮在 `<bili-comment-action-buttons-renderer>` 的第二层 shadow 内，完全够不到 |
| 递归函数 `hideInside(root)` 穿透所有 `el.shadowRoot` | 文本节点（如 "9小时前"）不是元素节点，`querySelectorAll` 找不到，无法隐藏 |
| `page.screenshot(clip={x,y,w,h})` 坐标裁剪 | `getBoundingClientRect()` 坐标与 `page.screenshot()` 的 clip 参数坐标系不一致 |
| 临时设置 `el.style.height + overflow:hidden` 截断元素 | web component 宿主元素的 CSS 不裁剪 shadow DOM 渲染内容 |
| 按视觉范围+800px隐藏页面所有评论线程 | 多余策略，且可能误隐藏无关评论 |

### 解决
XTong 在浏览器 DevTools 中右键 `<bili-comment-renderer id="comment">` → "捕获节点屏幕截图"，发现该元素天然只包裹置顶评论卡片，子评论在其外部。最终代码仅两行：
```python
comment_el = shot_page.locator('#comment').first
await comment_el.screenshot(path=screenshot_path)
```

### 教训
- **先验证 DOM 边界再写代码**。DevTools "捕获节点屏幕截图" 比盲写 JS 高效百倍
- B站评论区是 `<bili-comment-renderer>` 包裹 `<bili-comment-thread-renderer>` 的嵌套结构，sub-comment 是独立元素

## 2. B站动态发布 API

### 图片上传 412 错误
`POST /x/dynamic/feed/draw/upload_bfs` 返回 HTTP 412 + HTML。

**原因**：缺少 `User-Agent` 请求头，B站 API 必须模拟浏览器。

**解决**：添加 `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ...` UA。

### 图片上传 -101 错误
`code=-101 msg=账号未登录`。

**原因**：Cookie 过期，或账号未实名。

**解决**：重新运行 `get_cookies.py` 刷新 Cookie；B站发动态需要账号完成实名认证。

### 网页链接 type 踩坑
在 `content.contents[]` 中使用 `type: 4` → 发布后显示为"投票"。使用 `type: 5` → 显示为纯文本"网页链接"，不可点击。

**正确方式**：网页链接应使用 `attach_card` 结构（独立链接卡片），或将完整 URL 写在 `type: 1` 纯文本中让 B站自动识别。

```json
// 可行方案：URL直接写在正文
{"raw_text": "标题文字 https://t.bilibili.com/{dyn_id}?comment_on=1", "type": 1, "biz_id": ""}

// 备选：attach_card 链接卡片（独立于正文和图片）
"attach_card": {"type": 2, "web_card": {"url": "...", "title": "点击跳转"}}
```

### 发布动态必须参数
- `csrf`：URL 参数，值为 Cookie 中的 `bili_jct`
- `platform=web`：URL 参数
- `upload_id`：JSON body 中 `{mid}_{timestamp}_{4位随机数}`
- `meta.app_meta.from`: `"create.dynamic.web"`

## 3. Python 版本兼容

### `dict | None` 语法不兼容 Python 3.9
```python
# 3.10+ 可用，3.9 报错 TypeError
async def upload_image(...) -> dict | None:

# 兼容写法
from typing import Optional
async def upload_image(...) -> Optional[dict]:
```

服务端 Rocky Linux 9.6 自带 Python 3.9，本地 Windows 可能用 3.13。写代码时注意用 `Optional` / `Union` 等兼容语法。

## 5. B站风控

- 短时间内多次调用发布 API 并改变内容格式 → 触发实名验证
- 小号可能未实名，需用大号 Cookie
- 生产环境正常触发（每次置顶评论变更只发一条）不会触发风控

## 6. 截图高DPI最佳实践

```python
# 独立高DPI context，布局不变分辨率翻倍
shot_ctx = await self.browser.new_context(device_scale_factor=2)
main_cookies = await self.context.cookies()
await shot_ctx.add_cookies(main_cookies)
shot_page = await shot_ctx.new_page()
await shot_page.set_viewport_size({"width": 1080, "height": 1920})
# ... 截图后 close context
```

- 必须复制主 context 的 cookies，否则 B站因未登录跳转导致导航失败
- 用 `load` + `networkidle` 等页面稳定，`_safe_evaluate` 包装避免跳转残留
- 隐藏 B站固定导航栏（`.bili-header` 等 + `position:fixed` + `z-index>100`）
